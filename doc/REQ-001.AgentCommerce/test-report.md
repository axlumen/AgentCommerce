## 测试中发现并修复的 Bug

| Bug | 根因 | 修复 |
|-----|------|------|
| 注册/商品/订单创建 500 | `created_at = Column(DateTime, server_default=func.now())` 在 MySQL 中 refresh 后返回 None | 在 service 层显式设置 `created_at=datetime.now()`，schema 层 `created_at` 改为 Optional |
| admin 用户无权限 | admin 注册时未设置 `is_admin=True` | 脚本中添加 admin 用户初始化逻辑 |
| 空购物车下单返回 404 | `NotFoundError("购物车为空")` 映射为 404 | 行为符合设计，更新测试预期 |
| Button asChild 点击失效 | `<span>` 包装导致子元素事件丢失 | 改用 @base-ui/react 的 `render` prop |
| useEffect 闭包陈旧 | 空依赖数组 `[]` 导致状态不更新 | 补全依赖数组 |

### 注册按钮专项验证

**问题报告**: 用户反馈"注册按钮无法点击"

**排查过程**:
1. Playwright 检查 DOM：按钮渲染正常，未 disabled，type="submit"
2. 检查事件：form onSubmit 正确触发
3. 检查 API：`/api/auth/register` 被调用
4. 检查响应：返回 `400: 用户名已存在`

**根因**: 测试用户名已被注册，后端返回 400 错误，非按钮失效。

**验证**: 使用全新用户名 `brandnewuser` 注册成功（id: 14）。

**结论**: 注册按钮功能正常，错误提示 toast 表现符合预期。
